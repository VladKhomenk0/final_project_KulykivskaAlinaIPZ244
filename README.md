# Пам'ятайка (Електронний записник з нагадуваннями)

**Пам'ятайка** — це сучасний десктопний застосунок для персонального менеджменту нотаток, розроблений на платформі WPF (.NET 10). Додаток поєднує в собі гнучкість налаштувань, інтуїтивний інтерфейс та надійність збереження даних. Архітектура побудована на суворому дотриманні патерну MVVM та принципів чистого коду. Ця програма створена для того, щоб допомогти швидко організувати думки, не губити важливі дедлайни та мати під рукою надійний інструмент для планування.

## Основний функціонал

*   **Інтелектуальне керування:** створення, редагування та миттєве видалення нотаток.
*   **Система нагадувань:** стабільна робота фонового сервісу для відстеження важливих подій з візуальним сповіщенням.
*   **Гнучка візуалізація:** 
    *   Персоналізація через кольорові мітки (смужки-індикатори).
    *   Закріплення важливих записів (Pinned Notes) у топі списку.
    *   Динамічна зміна тем (Денна/Нічна) з автоматичним збереженням вибору.
*   **Розширений пошук:** фільтрація за змістом, категоріями та рівнями пріоритету в реальному часі.
*   **Надійність даних:**
    *   Багаторівнева система скасування та повторення дій (Undo/Redo).
    *   Функція швидкого дублювання нотаток одним кліком.
    *   Механізм створення резервних копій (Backup).
*   **Аналітика:** глибокий аналіз кожної нотатки (кількість слів, символів та час читання).

## Programming Principles

У проекті реалізовано наступні засади розробки:

1.  **SRP (Single Responsibility Principle):** Кожен модуль виконує одну чітку функцію. Наприклад, `SearchService.cs` відповідає виключно за алгоритми фільтрації.
2.  **DIP (Dependency Inversion Principle):** Високорівневі компоненти (ViewModels) взаємодіють з сервісами через інтерфейси (`INoteRepository`), що робить систему гнучкою до заміни джерела даних.
3.  **DRY (Don't Repeat Yourself):** Повторювана логіка сповіщень та валідації винесена у базовий клас `ViewModelBase.cs`.
4.  **SoC (Separation of Concerns):** Проект розділений на рівні: `Core`, `Infrastructure` та `UI`, які мають мінімальні зв'язки між собою.
5.  **KISS (Keep It Simple, Stupid):** Використання прямої роботи з файлами JSON для збереження даних без складних ORM-систем.

## Design Patterns

Проєкт базується на використанні архітектурних та поведінкових патернів:

1.  **Command Pattern (Команда):**
    *   **Реалізація:** [`UndoableActionCommands.cs`](./ElectronicNotepad.UI/Utils/UndoableActionCommands.cs)
    *   **Призначення:** Використовується для реалізації Undo/Redo. Кожна дія (додавання, видалення, оновлення) інкапсулюється в об'єкт-команду, що дозволяє легко керувати історією змін.
2.  **Repository Pattern (Репозиторій):**
    *   **Реалізація:** [`INoteRepository.cs`](./ElectronicNotepad.Core/Interfaces/INoteRepository.cs), [`NoteRepository.cs`](./ElectronicNotepad.Infrastructure/Repositories/NoteRepository.cs)
    *   **Призначення:** Ізолює логіку роботи зі сховищем даних від бізнес-логіки. Це дозволяє змінювати спосіб зберігання даних (файл/БД), не впливаючи на інтерфейс користувача.
3.  **MVVM (Model-View-ViewModel):**
    *   **Реалізація:** [`Note.cs`](./ElectronicNotepad.Core/Models/Note.cs), [`MainViewModel.cs`](./ElectronicNotepad.UI/ViewModels/MainViewModel.cs), [`MainWindow.xaml`](./ElectronicNotepad.UI/Views/MainWindow.xaml)
    *   **Призначення:** Основний архітектурний патерн, що забезпечує незалежність UI від логіки через Data Binding.

## Refactoring Techniques

Під час розробки були застосовані такі техніки рефакторингу:

1.  **Extract Class (Виділення класу):** Логіку фільтрації винесено в `SearchService.cs`, а роботу з темами — у `ThemeService.cs`.
2.  **Substitute Algorithm (Заміна алгоритму):** Система нагадувань переведена на фоновий потік (`Task.Run`) для стабільної роботи без фризів інтерфейсу.
3.  **Encapsulate Field (Інкапсуляція поля):** Дані моделей приховані за властивостями з контролем доступу для коректної роботи Binding.
4.  **Rename Method (Перейменування):** Назви методів (наприклад, `DuplicateNote`, `AddNoteWithUndo`) приведені до самодокументованого вигляду.
5.  **Simplify Conditional Expressions (Спрощення умов):** Використання LINQ у методах фільтрації дозволило замінити складні конструкції `if/else` на лаконічні запити.
